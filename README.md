# 🐢 每日海龟汤挑战

一款精心打造的**横向思维推理游戏**单页应用。每天一道海龟汤谜题，玩家通过提问推理出隐藏的真相。

> 海龟汤（Lateral Thinking Puzzle）是一种经典的推理游戏形式：出题者给出一个看似荒诞的情景（汤面），玩家通过提出是/否问题逐步逼近真相（汤底）。

## 预览

纯前端单文件应用，无需后端服务。打开 `index.html` 即可运行。

## 功能特性

### 🎮 游戏核心
- **每日谜题轮换** — 根据日期自动切换当日谜题
- **智能问答系统** — 基于规则引擎判定玩家提问（是/否/无关）
- **答案相似度评分** — 中文文本匹配算法，提交答案后显示接近度
- **提示系统** — 每题 3 条渐进式提示，使用提示会扣减得分
- **计时与统计** — 记录解题时间、提问次数、连续挑战天数

### 📊 排行榜
- 最快解出 / 提问最少 / 最佳推理 三个维度排名
- 本地模拟排行数据

### 📚 历史谜题库
- 所有谜题归档浏览，标注已解/未挑战状态
- 支持回放任意历史谜题

### ✏️ 玩家投稿
- 提交自创海龟汤谜题（汤名/汤面/汤底/关键线索）
- 提交前预览，投稿后可追踪审核状态
- 内置 AI 审核引擎自动评分

### 🔐 用户系统
- 注册 / 登录，数据按用户隔离存储
- 用户角色区分（普通用户 / 管理员）
- 游戏进度、成就、统计数据持久化

### 🛠 管理员后台
五个功能标签页：

| 标签 | 功能 |
|------|------|
| 仪表盘 | 用户数、谜题数、投稿数、通过率等统计卡片 + 快速操作 |
| 谜题管理 | 搜索、分页浏览所有谜题；新建 / 编辑 / 删除；内置谜题软删除 |
| 投稿审核 | 查看玩家投稿，通过 / 驳回 / 编辑后通过 |
| AI 创作 | 提示词驱动的谜题生成、迭代反馈、历史今日推荐、待发稿队列 |
| 用户管理 | 用户列表搜索、角色变更、统计查看、账号删除 |

### 🤖 AI 创作工作台（管理员）
- **提示词生成** — 输入关键词 / 故事原型，AI 基于模板引擎生成谜题
- **12 个快捷主题标签** + **6 种风格基调**选择
- **灵感骰子** — 随机组合人物 × 场景 × 反转 × 情感
- **迭代反馈** — 输入修改意见（如"让故事更阴暗""换一个场景"），AI 重新生成
- **历史上的今天** — 根据当日日期推荐节日、历史事件、著名案件等作为创作灵感
- **待发稿区** — 设置发布日期，管理发布队列
- **三态生命周期** — 工作区 → 待发布 → 已发布

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/你的用户名/daily-turtle-soup.git
cd daily-turtle-soup

# 方式一：直接打开
open index.html

# 方式二：本地服务器（推荐，避免部分浏览器 localStorage 限制）
python3 -m http.server 8080
# 访问 http://localhost:8080
```

## 部署到 GitHub Pages

1. 将仓库推送到 GitHub
2. 进入仓库 Settings → Pages
3. Source 选择 `main` 分支，目录选 `/ (root)`
4. 保存后等待部署完成，访问 `https://你的用户名.github.io/daily-turtle-soup/`

## 数据说明

所有数据存储在浏览器 `localStorage` 中：

| Key | 内容 |
|-----|------|
| `turtle_soup_users` | 用户账号列表 |
| `turtle_soup_global` | 投稿、AI 草稿、自定义谜题等共享数据 |
| `turtle_soup_v3_用户名` | 各用户的游戏进度、统计、成就 |

清除浏览器数据会重置所有进度。首次启动时会自动创建默认管理员账号。
